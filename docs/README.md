# docs/

Decision records and design notes. The persistent project brain is `CLAUDE.md`;
this directory holds the deeper material it would bloat by inlining.

## Convention

- One doc per decision / system / investigation, organized **by concern**, not
  chronologically.
- **Measurement-first:** prefer before/after data and reproduction steps over
  assertions.
- Link decisions to GitHub issues by number; flag open questions explicitly so they
  can be scaffolded into issues.
- Keep docs scannable: TL;DR at the top, tables over prose where it helps.
